# Day 4 总体验证报告

## 第四天目标

第四天目标是把循证问答接入第一版 NSCLC 研发证据查询闭环，并保持可追溯、安全、可回退：

- 支持基于本地已核验证据的问答。
- 可选使用 DeepSeek 组织答案。
- 保证先检索、后生成、再校验引用。
- 在 Web 页面展示回答、引用、限制说明和白盒过程。
- 与旧智能问答、旧白盒和旧 SQLite/Chroma 主链路隔离。

## 隔离关系

新循证问答只接入“研发证据查询”页面内部的“循证问答”页签。

它不复用旧智能问答、旧白盒溯源或旧 SQLite/Chroma 问答主链路，也不使用旧页面的自由生成逻辑。旧页面保留原状。

## 分类与安全规则

循证问答先执行问题分类和安全规则，再进入检索。

支持问题类型：

- 来源检索
- 试验状态
- 证据链
- 监管状态
- 企业对比
- 证据缺口
- 安全边界

禁止问题在证据检索前拦截，包括诊断、个体治疗或用药建议、疗效保证、跨试验疗效排名、成功率预测、投资建议和企业综合排名。

安全拦截结果不检索证据、不调用模型、不生成引用，并显示中文安全类别。

## 证据检索与证据包

本地证据检索复用：

- `SourceRegistryService`
- `EvidenceChainService`
- `CompanyEvidenceComparisonService`

证据包包含本次允许引用的 `source_id`、证据链、关联监管背景、企业对比结果、证据缺口和数据版本。模型只能看到本次证据包，不能新增未检索来源。

## DeepSeek 延迟初始化

DeepSeek 客户端只在以下条件同时满足时延迟创建：

- 请求为 `auto` 模式。
- 已配置本地 DeepSeek 环境变量。
- 问题通过安全规则。
- 本次检索存在可用证据。

`local` 模式不创建模型客户端。无密钥、模型异常、超时、鉴权失败、额度不可用、输出非法或引用校验失败时，自动回退本地结构化摘要。

## 生成与引用校验

循证问答执行顺序固定为：

```text
用户问题
→ 安全规则
→ 本地证据检索
→ DeepSeek组织答案
→ 引用校验
→ 返回回答、引用、限制和白盒过程
```

引用必须通过本地来源登记校验：

- `source_id` 必须属于本次检索证据包。
- `source_id` 必须存在于本地来源登记表。
- `source_url` 以本地登记表为准。
- 重复引用会去重。
- 无效引用会被移除。
- 如果模型没有留下有效引用，则回退本地摘要。

## auto/local 与失败回退

- `auto`：DeepSeek 可用时组织答案；失败时回退本地摘要。
- `local`：只使用本地结构化证据，不调用模型。

API metadata 和前端状态以本次实际执行结果为准，不使用 capabilities 中的配置模型冒充本次实际模型。

## API 与 Web 页面

新增 API：

- `GET /api/evidence/grounded-qa/capabilities`
- `POST /api/evidence/grounded-qa`

Web 页面位于“研发证据查询”第四个页签，与来源检索、证据链、企业对比共享同一证据口径。

页面展示：

- 回答
- 引用资料
- 限制说明
- 安全提示
- 白盒过程
- 证据链跳转

## B015/B016 真实 API 冒烟结果

真实 API 冒烟由用户在本地完成。结果显示：

- `generation_mode_used=llm`
- `llm_used=true`
- `fallback_used=false`
- 使用配置模型组织答案。
- 引用仅包含本次检索允许的 B015/B016。
- 未出现虚构来源、密钥泄露或引用 URL 偏离本地登记表。

本文档不记录真实密钥、余额、请求头或本地个人路径。

## RATIONALE-315 Web 验收结果

浏览器验收确认：

- B011、B012、B013 属于同一项试验的试验证据。
- B016 只作为关联监管背景。
- B016 不计入试验证据数量。

- 证据缺口使用“当前收录样本中”限定，不推断全球范围不存在资料。

## 个体治疗问题修复

浏览器验收曾发现个体治疗建议问题被误判为来源检索，并显示本地证据摘要和配置模型名称。

修复后：

- 命中 `prohibited_or_unsupported`。
- 安全类别为“个体治疗或用药建议”。
- 在检索前安全拦截。
- 不检索证据。
- 不调用 DeepSeek。
- 不返回引用。
- 前端显示“安全规则拦截”和“未调用模型”。
- 主回答、限制说明和安全提示不直接暴露内部 snake_case 类别代码。

## 数量拆分

证据链问题 trace 中拆分：

- `source_count`：检索来源总数。
- `trial_evidence_count`：试验证据数量。
- `related_regulatory_count`：关联监管背景数量。

非证据链问题不显示无意义的 0 值拆分。

## 当前限制

- 第一版仅覆盖当前冻结的 31 条人工核验 NSCLC 来源。
- 证据缺口只表示当前收录样本不足，不表示全球范围不存在相关资料。
- DeepSeek 仅负责组织答案，不负责新增事实或新增引用。
- 循证问答不是医学、投资或商业决策系统。
- 未建设面向全部疾病、全部企业和持续增量更新的大规模知识库。
