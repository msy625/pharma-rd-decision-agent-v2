# 证据规则

本文档定义“药研制策”初版的证据等级、证据冲突和风险提示规则。规则适用于已冻结的第一版数据范围：NSCLC、恒瑞医药、百济神州（BeOne Medicines，原BeiGene）和 31 条人工核验来源。

初版只使用分类等级，不使用 random、固定随机评分、MOCK 数据或没有依据的 0-100 分。证据强度不能由来源数量简单相加得到；必须结合结论类型、来源优先级、证据版本、时间和冲突情况判断。

## 1. 来源类型

| 类型 | 定义 | 第一版示例 |
|---|---|---|
| `REGULATORY_PRIMARY` | 监管机构原始页面、正式产品信息、正式批准文件。 | EMA EPAR Tevimbra（B015）；EMA/CHMP 文件（B016）。 |
| `TRIAL_REGISTRY` | ClinicalTrials.gov 或其他正式临床试验登记。 | H003-H007、B003-B005、B012。 |
| `PEER_REVIEWED_PUBLICATION` | PubMed 收录的同行评议论文。 | H008-H012、B006-B011。 |
| `COMPANY_FORMAL_DISCLOSURE` | 公司正式公告、年度报告、投资者材料。 | H002、H013、H014、H015、B002。 |
| `COMPANY_PIPELINE` | 公司当前研发管线图或管线页面。 | B014。 |
| `COMPANY_TRIAL_PAGE` | 公司临床试验页面。 | B001、B013。 |
| `SECONDARY_SOURCE` | 新闻、媒体、行业数据库或二手整理资料。 | 第一版未核验，不进入主要结论。 |

`SECONDARY_SOURCE` 当前不得用于支撑主要结论；如后续补充，必须明确来源、日期、二手属性和可替代的原始来源缺口。

## 2. 按结论类型选择来源

不建立单一绝对证据排名。不同结论类型采用不同优先来源：

| 结论类型 | 优先来源 | 辅助来源 | 注意事项 |
|---|---|---|---|
| 监管状态 | `REGULATORY_PRIMARY` | 公司正式公告 | 公司公告只能辅助支持，不能替代监管原始来源。 |
| 临床试验当前状态 | `TRIAL_REGISTRY` | 公司试验页面、论文 | 论文中的试验状态只能表示论文发表时状态。 |
| 试验设计、阶段、入排标准 | `TRIAL_REGISTRY` | 同行评议论文、公司试验页面 | 应保留 NCT 编号、正式 Study ID 和入排标准。 |
| 研究结果和主要终点 | 最终同行评议论文、监管评审资料 | 中期同行评议论文 | 公司新闻稿不得替代完整论文结果。 |
| 公司当前研发管线 | `COMPANY_PIPELINE` | 年报、投资者材料 | 管线图未列出某项目不能直接证明项目终止。 |
| 公司研发投入和整体战略 | 年度报告、公司正式披露 | 投资者材料 | 不把战略表述写成临床结论。 |
| 名称、别名和研发代号 | 多个独立来源交叉确认 | 公司与登记来源 | 保留原始名称，统一标准化名称。 |

## 3. 证据强度

| 等级 | 判定规则 | 输出要求 |
|---|---|---|
| `STRONG` | 结论得到对应类型的权威原始来源直接支持；或同时获得权威来源与独立来源一致支持；监管批准结论必须有监管机构当前正式页面支持。 | 可以作为主要结论，但仍需列出来源和日期。 |
| `MODERATE` | 有一个对应类型的可靠原始来源；或两个来源相互支持但尚缺更高优先级来源；来源不存在实质冲突。 | 可以回答，但需提示更高优先级来源是否缺失。 |
| `WEAK` | 只有公司单一来源；只有中期结果且无最终分析；缺少独立来源；来源时间较旧或范围受限。 | 只能作为观察或线索，不应写成确定性结论。 |
| `INSUFFICIENT` | 缺少必要来源；存在尚未解决的实质冲突；关键字段缺失；只能依赖推断；无法可靠回答。 | 应返回证据不足，并列出缺口和最小补充办法。 |

重要限制：
- 不得通过简单计算来源数量自动判定 `STRONG`。
- 多篇论文属于同一试验时，不能当作多个独立试验证据。
- `HIGH` 风险提示不代表药物危险，只代表当前证据不足以支持可靠结论。

## 4. 证据版本规则

| 版本 | 含义 | 处理规则 |
|---|---|---|
| `FINAL` | 最终分析或最终主要结果。 | 优先于同一试验的 `INTERIM`。 |
| `INTERIM` | 中期分析。 | 保留，但不能覆盖最终分析。 |
| `LONG_TERM_FOLLOWUP` | 长期随访或长期结果。 | 用于补充长期信息，不自动覆盖所有最终分析字段。 |
| `REGULATORY_CURRENT` | 当前监管正式状态。 | 用于回答当前授权或批准状态。 |
| `REGULATORY_OPINION` | 监管积极意见或评审意见。 | 不等于最终批准。 |
| `CURRENT_REGISTRY` | 当前试验登记状态。 | 用于回答当前 ClinicalTrials.gov 状态。 |
| `COMPANY_PIPELINE_SNAPSHOT` | 特定日期的公司管线快照。 | 仅说明该日期管线图展示范围。 |

同一试验存在多个证据版本时，必须保留历史版本，不得直接删除旧证据。建议字段包括：
- `evidence_version`
- `analysis_stage`
- `supersedes_source_id`
- `is_latest_evidence`
- `effective_date`
- `data_cutoff_date`

## 5. 冲突类型

| 类型 | 定义 | 示例处理 |
|---|---|---|
| `ALIAS_MISMATCH` | 中文名、英文名或研发代号没有正确归一化。 | 先查别名表和原始标题。 |
| `STATUS_TIME_DIFFERENCE` | 不同日期来源反映不同试验状态。 | 以当前登记或当前监管页面说明当前状态，同时保留历史来源。 |
| `PHASE_MISMATCH` | 临床阶段描述不一致。 | 检查来源日期、适应症、试验编号和是否为不同队列。 |
| `POPULATION_MISMATCH` | 研究人群、组织学类型或治疗线次不一致。 | 不合并为全部 NSCLC，按人群拆分。 |
| `INDICATION_MISMATCH` | 适应症范围不一致。 | 检查监管适应症、试验入组和论文人群是否对应。 |
| `REGULATORY_STATUS_MISMATCH` | 积极意见、提交申请、受理、批准等状态被混用。 | 区分 `REGULATORY_OPINION` 与 `REGULATORY_CURRENT`。 |
| `EVIDENCE_VERSION_DIFFERENCE` | 中期、最终、长期随访版本被混用。 | 建立版本链，不重复计数。 |
| `SOURCE_SCOPE_DIFFERENCE` | 来源展示范围不同，例如当前管线图未展示已完成项目。 | 标记为范围差异，不直接判定冲突。 |
| `UNRESOLVED_CONFLICT` | 经过名称、日期、范围和版本检查后仍无法解释的冲突。 | 保留冲突并在回答中说明。 |

## 6. 冲突处理顺序

1. 检查企业、药物和研究名称是否正确归一化。
2. 检查来源日期和数据截止日期。
3. 检查是否为中期、最终、长期随访或监管事件链。
4. 检查来源展示范围是否不同。
5. 根据结论类型选择对应的优先来源。
6. 保留低优先级来源及历史记录。
7. 无法解决时标记 `UNRESOLVED_CONFLICT`。
8. 不得静默覆盖冲突字段。
9. 系统输出必须说明冲突及采用某个来源的理由。

## 7. 风险提示代码

| code | name_zh | description | trigger_condition | user_message | severity |
|---|---|---|---|---|---|
| `R01_MISSING_REGULATORY_PRIMARY` | 缺少独立监管原始记录 | 缺少监管机构原始页面、正式产品信息或批准文件。 | 监管状态结论没有 `REGULATORY_PRIMARY` 支持。 | 当前缺少独立监管原始记录，该监管状态只能作为待补充信息。 | `HIGH` |
| `R02_COMPANY_ONLY` | 仅公司来源支持 | 结论只由公司公告、年报、投资者材料或公司页面支持。 | 支持来源全部为公司来源，缺少登记、论文或监管来源。 | 当前结论只有公司来源支持，需要独立来源交叉验证。 | `CAUTION` |
| `R03_INTERIM_ONLY` | 仅中期分析 | 目前只有中期结果或中期论文。 | `analysis_stage` 包含中期，且无最终版本或监管结果支持。 | 当前只有中期分析，不能视为最终结果。 | `CAUTION` |
| `R04_OUTDATED_SOURCE` | 来源可能过时 | 来源日期较旧或已有更新版本。 | 存在更新的 `source_last_updated`、`online_publication_date` 或 `supersedes_source_id`。 | 当前来源可能不是最新版本，请优先查看更新来源。 | `CAUTION` |
| `R05_STATUS_TIME_DIFFERENCE` | 不同日期状态不同 | 不同来源日期对应不同状态。 | 同一项目状态字段不一致且可由日期解释。 | 不同日期来源显示的状态不同，回答采用当前优先来源并保留历史记录。 | `INFO` |
| `R06_POPULATION_NOT_ALIGNED` | 研究人群不一致 | 比较对象的人群、组织学类型或治疗线次不同。 | 比较中 `population`、`histology` 或 `treatment_line` 不一致。 | 比较对象研究人群不一致，不能直接外推为同一人群结论。 | `CAUTION` |
| `R07_CROSS_TRIAL_COMPARISON` | 跨试验比较 | 正在比较不同试验的结果。 | 回答涉及不同 `parent_trial_id` 或不同研究设计的结果比较。 | 当前是跨试验比较，不能直接进行疗效排名。 | `HIGH` |
| `R08_DUPLICATE_EVIDENCE` | 重复计数风险 | 同一试验多篇论文或多来源可能被重复计数。 | 多个来源共享同一 `parent_trial_id` 或证据关系。 | 多个来源对应同一试验，不能重复计为多个独立试验。 | `CAUTION` |
| `R09_MISSING_CRITICAL_FIELD` | 关键字段缺失 | 靶点、阶段、人群、状态或来源日期等关键字段缺失。 | 必要字段为空、`来源未明确` 或 `文件未明确`。 | 当前关键字段缺失，不能可靠回答该部分。 | `HIGH` |
| `R10_UNRESOLVED_CONFLICT` | 未解决冲突 | 存在无法解释的实质冲突。 | 冲突类型为 `UNRESOLVED_CONFLICT`。 | 当前来源存在未解决冲突，不能生成确定结论。 | `HIGH` |
| `R11_SOURCE_SCOPE_LIMITATION` | 来源范围受限 | 来源只展示部分项目或特定时间点。 | 来源为管线快照、单页材料或备注含范围限制。 | 当前来源展示范围有限，不能证明未展示项目不存在或已终止。 | `CAUTION` |
| `R12_NO_DIRECT_COMPARATIVE_TRIAL` | 缺少头对头试验 | 两家公司项目之间没有直接比较试验。 | 企业比较问题没有同一试验或头对头设计支持。 | 当前没有直接头对头临床试验，不能做直接疗效优劣判断。 | `HIGH` |

## 8. 系统回答格式

所有重要回答至少包含：

```text
结论：
研究范围：
当前状态：
证据强度：
主要证据：
证据版本：
风险提示：
数据截至：
数据缺失：
限制说明：
```

来源必须能够追溯到 `source_id`、PMID、NCT 编号或监管链接。回答中如采用某一优先来源，应说明采用理由。

## 9. 禁止输出

- 个体治疗选择。
- 疾病诊断。
- 药物疗效保证。
- 跨试验直接疗效排名。
- 公司投资价值判断。
- 无来源支持的未来成功概率预测。
- 使用 random、MOCK 数据或无依据的固定数值评分。
- 将公司新闻稿替代完整论文结果。
- 将监管积极意见写成最终批准。
- 将来源核验状态写成临床研究状态。

## 10. 真实案例验证

### 案例 1：NCT04619433

- 涉及来源：H006。
- 规则判断：`TRIAL_REGISTRY` 是临床试验当前状态的优先来源。
- 验证结果：当前研究状态为 `Terminated`，来源核验状态为“已人工核验”。“研究终止”和“来源有效”不是一回事。
- 风险提示：若用户把“已人工核验”理解为“研究进行中”，触发 `R05_STATUS_TIME_DIFFERENCE` 或状态字段误用提示。

### 案例 2：RATIONALE-304

- 涉及来源：B006、B007。
- 规则判断：B006 为中期分析，B007 为最终分析；`FINAL` 优先于 `INTERIM`。
- 验证结果：B007 是当前主要结果版本；B006 和 B007 仍属于同一试验 `NCT03663205`，不能计为两项独立试验。
- 风险提示：触发 `R08_DUPLICATE_EVIDENCE`，中期版本可触发 `R03_INTERIM_ONLY`，但有最终版本时不应只按中期回答。

### 案例 3：RATIONALE-315

- 涉及来源：B011、B012、B013。
- 规则判断：B012 是 `TRIAL_REGISTRY` 主要登记，B011 是论文，B013 是公司试验页面。
- 验证结果：三条来源均关联 `NCT04379635`，只对应一项 RATIONALE-315 试验。
- 风险提示：触发 `R08_DUPLICATE_EVIDENCE`，防止重复计数。

### 案例 4：B014 当前管线图

- 涉及来源：B014。
- 规则判断：B014 是 `COMPANY_PIPELINE` 和 `COMPANY_PIPELINE_SNAPSHOT`，数据截止日期为 2026-05-01。
- 验证结果：未列出 RATIONALE-315 应判断为 `SOURCE_SCOPE_DIFFERENCE`，不得判断为项目不存在或已终止。
- 风险提示：触发 `R11_SOURCE_SCOPE_LIMITATION`。

### 案例 5：B015 与 B016

- 涉及来源：B015、B016。
- 规则判断：B016 为 `REGULATORY_OPINION`，B015 为 `REGULATORY_CURRENT`。
- 验证结果：两者属于监管事件链，不是冲突。当前正式授权状态应以 B015 的 EMA EPAR 页面为优先来源。
- 风险提示：若把 B016 写成最终批准，触发 `REGULATORY_STATUS_MISMATCH` 和 `R10_UNRESOLVED_CONFLICT` 的人工复核流程。

### 案例 6：H013

- 涉及来源：H013。
- 规则判断：H013 未明确 SHR-A1811 在 NSCLC 中的具体临床阶段。
- 验证结果：应触发 `R09_MISSING_CRITICAL_FIELD`；不得从其他无关联字段推断具体临床期数。
- 风险提示：若需要回答 SHR-A1811 在该材料中的 NSCLC 具体阶段，应返回“文件未明确”。
